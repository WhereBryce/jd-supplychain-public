仓配费节降通用框架

把两张 HC 结构图抽象成跨品类可复用的骨架：仓配费 = 非异常(基础) + 异常(拆单/代发)，按 费项 → 方式 → 抓手 三层拆解；轻货仓 / 城市仓是横跨多处的枢纽抓手。 HC 只是往骨架上贴了「重赠品 + 多小件」这层皮——换品类时骨架不动，只需重估主攻方向、替换选品参数。

非异常侧管「基础计费」，异常侧管「拆单/代发多花的钱」，合起来是一张完整仓配费地图。

这张地图怎么读
  • 横向两大块：非异常是「订单本身就要付的基础费」，异常是「因为货没放对地方而多付的钱」——异常的每一项都是可以被消灭的，非异常只能被优化。
  • 末端即计费口径：出库生产费永远按子单件数、配送费永远按首续重×线路；异常无非是把这两条口径在「多子单/更长链路」上又跑了一遍。
  • 两处计费差异要记牢：拆单的出库 upcharge 是每子单各计一个首件；异常配送里跨区平行=子单分开算、同区代发=并父单按最贵算——同样是拆，计费规则不同。

二结构骨架：费项 → 方式 → 抓手

一句话记住 3 费项 · 4 方式 · 2 枢纽仓：出库费靠降件数；配送费靠就近履约 + 换便宜通道；异常费靠把货放对；轻货仓 / 城市仓是横跨多处的枢纽抓手。

三层怎么读：

  • 费项（花在哪）：出库生产费 / 配送费 / 异常费——对应 JD 真实计费科目。
  • 方式（往哪使劲）：降件数 / 就近履约 / 更廉价运输 / 库存左移。方式层按需——只有配送费需要分两方式，出库费、异常费方式单一，费项直连抓手。
  • 抓手（「杠杆」·可直接执行）：每个方式名下的具体动作，详见第四节。
  • 枢纽抓手：轻货仓 / 城市仓被「更廉价运输 + 库存左移选品下沉」多处复用，一个动作同时压配送费、异常费、出库费——最高效的着力点。

三两仓网枢纽详解：轻货仓 vs 城市仓

两者都能把仓配费压到主仓的 ~50%，但设计目的、覆盖、时效完全不同。

轻货仓城市仓
数量 / 位置6 个：北、上、广、成都、西安、武汉3 个：北、上、广
为谁设计小件赠品专仓一线城市爆品专仓
为何便宜（~50%）降级时效，出库生产费 + 配送费都更低出库不过分拣中心、直达派送站点，链路更短
时效更慢（以时效换低价）更快（链路短，快速触达消费者）
边界 / 代价时效降级辐射范围小，仅北上广核心城区
选品宝洁赠品进仓只选适配当地偏好的爆品

四各费项的方式与抓手

费项 1 · 出库生产费 ── 方式：降件数

按子单核算：首件 + (件数−1) × 续件（A 件型 首件 2.8 / 续 0.5；B 件型 首件 3.5 / 续 0.6）。降子单内件数 = 降拣货次数。抓手（3 种「产品机制适配」）：

  1. 大包装新品 — 一次买多件/套的，多「件」合成 1「件」。
  2. 赠品打包 — 绑赠里减赠品数量，或出厂把多个赠品预打包成一件。
  3. 主+赠组合装（最激进）— 对高频一起买、或品牌有意为之的主+赠，直接做成一个包装的新品，一步消除多件。

费项 2 · 配送费 ── 两个并列方式

按重量 × 路线报价，两个方式各降不同因子：

方式 A · 就近履约（降距离）

  • 抓手 库存下沉：让库存更近消费者。关键：JD 路由是整单优先，不能只下沉主品，绑送赠品也要一起下沉——否则赠品不在本地会触发跨区/拆单。

方式 B · 更廉价运输（降通道单价）

  • 抓手① 快运替快递：对大订单场域（企业购 / 批发）用快运，依赖 JD 系统为不同订单匹配更经济的履约方案。
  • 抓手② 降级时效仓型：走轻货仓 / 城市仓（~50% 主仓费）。

费项 3 · 异常费（拆单 / 代发）── 方式：库存左移（把货放对）

根因都是本地库存不足 / 不正确，父单被拆成多子单并发满足。下面两侧的抓手都指向「选品 + 库存正确 + 合仓」。

A｜出库生产费侧 — 拆单 5 类场景（每个子单单独计出库费）

#拆单场景解法
1跨区（11 大 RDC：传统 8 仓 + 南京/郑州/杭州）正确选品 + 库存
2同区跨城市正确选品 + 库存
3同区跨网络（主仓/轻货仓/城市仓属不同网）正确选品 + 库存
4同区同城跨不同仓同城合仓
5虚拟组套 + X（同仓也拆组套与 X）系统级，需 JD 干预
场景 1–3 靠「选品 + 库存正确」解决，占 ~90%；场景 5 是系统设计，占 ~10%；场景 4 靠同城合仓。

B｜配送费侧 — 两种异常履约（都因本地库存不足）

跨区平行履约同区代发
触发本地无法整单满足 → 跨区协助本地无法整单满足 → 同区跨城协助
链路变长，可能拆单变长，可能拆单
计费跨区子单与同区子单分开按各自首续重计费多子单合并到父单，按最贵首续重计费

五迁移用法（决策顺序）

  1. 先拆这个品类的仓配费：异常占比 vs 非异常占比（异常高 → 先做库存左移）。
  2. 非异常侧看件型分布：多小件 → 主攻降件数；多重货 → 主攻就近履约 / 换便宜通道。
  3. 落地优先用两个枢纽仓（轻货仓/城市仓），一个动作同时压配送费、异常费、出库费。
  4. 最后按品类替换选品参数与赠品策略。
