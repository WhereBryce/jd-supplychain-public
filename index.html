▶ 我要扔一份材料 先了解一下

01为什么要有这个知识库

团队的经验大多散落在聊天、邮件、脑子里，人一走就没了。这个知识库把它们变成可检索、可复用、可复利的团队资产。

🧠 单一事实来源

供应链的口径、流程、概念集中一处；不再「同名不同义」、各写各的。

🔁 越用越聪明

碎片 → 原子知识 → 洞察 → 反哺应用。知识之间建立连接，产生新观点。

🤝 人人可贡献

存在 SharePoint，团队协作编辑。零门槛投递，不用担心「写得对不对」。

02怎么贡献：直接把材料扔进 Inbox

Inbox 是无门槛落地区。任何格式、任何半成品，直接扔进去就行——不用判断价值、不用归类、也不用担心冲突。

1

打开 Inbox 文件夹

点下面的 SharePoint 链接，或用已同步到本地的 _Inbox/ 文件夹。

2

把材料拖进去

Word / PPT / PDF / Excel / 图片 / 截图 / 链接 / 一句话记事……任何格式都行，直接拖拽或上传。

3

（可选）留个线索

文件名带上你的名字或一句话主题最好，方便回溯；不写也没关系。就这样，完成！

🙌 随便扔，不会冲突。　你每次是往文件夹里新增一个文件，不是去改别人正在编辑的同一个文件，所以多人同时扔也不会产生「冲突副本」。放心投。

03知识库怎么运转（云 ⇅ 本地）

你扔进 SharePoint 的材料，会通过下面这条链路变成团队随手可查的知识：

你扔材料 ▶ SharePoint（云·共享） ⇅ 自动同步 本地文件夹 ▶ Obsidian + AI 整理 ▶ 体系化知识 / 洞察

☁️ SharePoint = 团队共享盘

整个知识库存在这里，人人可读写。你扔的材料先落在这，大家都能看到。

💻 本地 = 同步的镜像

装了 SharePoint/OneDrive 同步的电脑，会把库同步成一个本地文件夹，离线也能看，改动自动回传云端。

🔗 Obsidian = 阅读/编辑器

用 Obsidian 打开本地文件夹，就能看双链、关系图、领域地图（MOC）。纯 Markdown，无需插件。

🤖 AI = 整理与产洞察

Copilot 在本地对库做原子化、建关联、跑合成产洞察；结论再反哺领域与应用。

💡 只想贡献？　你完全不需要装 Obsidian——在 SharePoint 网页里把材料扔进 _Inbox 就够了。整理和沉淀由管家 + AI 完成。

04碎片如何变成知识（三个循环）

你贡献的碎片，会经过三个循环变成团队资产：

碎片/资料 ──①结构化──▶ 原子知识 ──②沉淀──▶ 洞察 ──③合成──▶ 反哺应用

① 结构化

碎片拆成「一念一笔」的原子知识，打标签、建双链、挂到领域地图 MOC。

② 沉淀

对话/思考里冒出的新观点，写成「洞察」：主张 + 推理 + 依据 + 置信度。

③ 合成

跨知识跑批，找矛盾 / 空白 / 类比，升级成新洞察，再反哺回领域。

05知识长什么样

库里的每篇笔记有个「类型」。你只要会投 Inbox 就够了，其余交给整理；了解结构有助于你读得更快。

类型含义放哪
atom 原子一个概念/事实，自足可复用领域文件夹
source 来源某份文档/书/视频的提炼领域文件夹
moc 地图某领域的入口与导航领域根 / 顶层
insight 洞察想出来的新观点Insights/
class 本体实体领域对象的显式定义（订单/库存/仓…）_Meta/Ontology/

🧩 本体 Ontology

供应链的语义层：订单、商品 SKU、客户、仓 RDC、库存、配额、KPI…及它们之间的关系。是上层应用的口径根基。

⚙️ 应用 Applications

围绕知识库构建的自动化：低库存告警、智能补货、现货率报告等。代码在独立 Git 仓库，消费本体口径。

06和 AI 协作：一句话触发

在 Copilot 里对着知识库说这些词，AI 就会动手：

整理收集箱 — 把 Inbox 碎片原子化、归类、清空 原子化 X — 把 X 拆成原子笔记 找关联 — 为当前笔记补双链与标签 合成 <域> — 对某领域跑批产洞察 沉淀 / 记一下 — 把刚才的观点写成洞察 处理图片 X — 给截图配文字注释让它可检索 巡检洞察 — 回顾洞察层，列「想向你要的信息」
🗓️ 管家轮值（Steward）：每周由一位同学执行「整理收集箱」，把大家投的碎片清零、沉淀入库。轮到你时，说一句「整理收集箱」即可。

07常见问题

我不确定这条信息对不对 / 有没有用，要投吗？

要。Inbox 就是收「不确定」的地方——它只是原料。价值判断交给整理环节，AI 会标为噪音待你确认，不会擅自删你的在制品。

能投什么格式？有限制吗？

没有限制。Word / PPT / PDF / Excel / 图片 / 截图 / 链接 / Markdown……任何格式直接扔进 _Inbox/。图片和文档在整理时会被配上可检索的文字注释，说「处理图片 X」即可。

大家一起扔，会不会冲突/覆盖别人的东西？

不会。你每次是往文件夹里新增一个文件，不是编辑别人正在改的同一个文件，所以多人同时扔也不会产生「冲突副本」。整理时也不会擅自删除任何人的在制品。

写笔记有格式要求吗？

投 Inbox 阶段：随意，一行也行。正式沉淀成知识时由整理者按《知识库约定 Conventions》规范化，你不用操心。
